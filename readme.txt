ИНСТРУКЦИЯ ПО ЗАПУСКУ ПРОЕКТА

1. Штатными средствами PostgreSQL создать базу данных 'smartsoft', для которой username - 'postgres', password - 'vital' . Если у базы данных другое название, логин, пароль, то в файле 'application.properties' следует изменить соответствующие значения параметров.

2. В этой базе данных создать таблицу gorod с помощью следующего скрипта:

CREATE TABLE "gorod" (
  "id" serial PRIMARY KEY,
  "ssoid" varchar(255) NOT NULL,
  "ts" integer,
  "grp" varchar(255) ,
  "type" varchar(255),
  "subtype" varchar(255),
  "url" varchar(255),
  "orgid" varchar(255),
  "formid" varchar(255),
  "code" varchar(255),
  "ltpa" varchar(255),
  "sudirresponse" varchar(255),
  "ymdh" varchar(50)
);

3. Распаковать проект в любую директорию и импортировать в IDE, например Spring Tool Suite.

4. Для запуска проекта запустить SmartsoftApplication.java как джава-приложение.

5. Поместить файл с данными 'test_case.csv' в папку %userprofile%. При изменении директории или названия файла следует внести соответствующие правки в метод populate() класса GorodDaoImplJdbc.

6. Открыть в браузере URL 'http://localhost:8080/'. Далее переходить по ссылкам.

2. Запросы к бд строились при определённых допущениях из-за отсутствия полной картины бизнес-процесса, а именно:

 Задание 1. Вывести список пользователей и используемых ими форм за последний час.

- в файле содержится выборка за один день 2017-07-11
- время в колонке ts измеряется в миллисекундах относительно начала рабочего дня 9:00, таким образом диапазон значений ts составляет около пяти часов.
- точка, относительно которой отсчитывается последний час, -- это максимальное значение поля ts. То есть, границами искомого диапазона являются 
[max_ts - 3'600'000 мс, max_ts].
------------------------------------------------------------

Задание 2. Вывести список пользователей, которые начали активность на форме и не дошли до конца. 

- предполагается, что для всех услуг grp начальным этапом запроса пользователя является start и конечным send
- один пользователь может делать несколько запросов по одному виду услуг. Запросы направляются последовательно. То есть, второй запрос может начаться после того, когда закончился первый. 
- если было несколько запросов по одному виду услуг, то все запросы, кроме быть может, последнего, являются оконченными, т.е. последний этап у них send
- неоконченным может быть только последний запрос в цепочке
- запрос считается неоконченным, если последним по времени ts этапом не является send
- запросы с пустым полем ssoid не исключались из рассмотрения
